G. Goals

G.1 Context and overall objectives

The "Maison Intelligente - Systeme d'alarme" is a security system designed to continuously monitor a home. Its primary objective is to detect specific critical events—intrusions, fires, or medical emergencies—and automatically alert the appropriate emergency services (police, fire department, ambulance). The system aims to ensure safety by providing 24/7 operational monitoring.

G.2 Current situation

G.3 Expected benefits

The deployment of this system is expected to provide the following benefits:
- Automated Emergency Response: Automatic calling of police, fire, or ambulance services based on the specific type of event detected.
- Continuous Protection: The system guarantees 100% uptime for constant protection.
- Visual Verification: Integration with security cameras allows for visual verification of events.
- Personal Safety: Specific support for medical emergencies (e.g., falls) via wearable controllers.

G.4 Functionality overview

The system performs the following high-level functions:
- Event Monitoring: Detects intrusions, fires, and medical emergencies.
- Alerting: Automatically contacts the relevant emergency service upon detection.
- Integration: Connects with smart home devices for automated responses and security cameras for verification.
- Multi-location Support: Supports monitoring for multiple homes or properties for a single user.

G.5 High-level usage scenarios

  • Scenario 1 (Medical): A user wearing the physical controller experiences a fall. The system detects the medical emergency and immediately alerts the ambulance service.
  • Scenario 2 (Intrusion): The system detects an intrusion. Within 10 seconds, the system alerts the police and uses security cameras for visual verification.

G.6 Limitations and exclusions

  • False Alarm Rate: The system must maintain a false alarm rate of less than 1% per year.

G.7 Stakeholders and requirements sources

The primary stakeholders for this project include:
- Homeowners
- Renters
- Emergency Services: Police, Fire Department, and Ambulance services

E. Environment

E.1 Glossary

[E.1.1] The following terms are essential to the project domain:
- Home: The physical location being monitored.
- SecuritySystem: The core system performing monitoring and alerting.
- Event: A critical occurrence requiring attention, specifically intrusions, fires, or medical emergencies.
- EmergencyService: External entities including Police, Fire Department, and Ambulance.
- SmartHomeDevice: External devices integrated for automated responses.
- SecurityCamera: Devices used for visual verification of events.
- User: The individual interacting with the system (Homeowner or Renter).

E.2 Components

The system environment includes the following external elements:
- [E.2.1] Smart Home Devices: Used for automated responses.
- [E.2.2] Security Cameras: Used for visual verification.
- [E.2.3] Emergency Services Infrastructure: The communication channels to Police, Fire, and Ambulance.

E.3 Constraints

The environment imposes the following strict constraints on the system:
- [E.3.1] Uptime: The system must maintain 100% uptime to ensure it is operational 24/7.
- [E.3.2] Response Time (Intrusion): Maximum 10 seconds to detect and alert.
- [E.3.3] Response Time (Fire): Maximum 5 seconds to detect and alert.
- [E.3.4] Response Time (Medical): Immediate response required.

E.4 Assumptions

  • [E.4.1] Assumption: Users with medical concerns will wear the physical controller (watch) to enable fall detection.

E.5 Effects

  • [E.5.1] The system will trigger actions in external Emergency Services (e.g., dispatching units).
  • [E.5.1] The system will trigger automated responses in connected Smart Home Devices.

E.6 Invariants

  • [E.6.1] The false alarm rate must remain below 1% annually.

S. System

S.1 Components

The system consists of the following major components:
- Central Monitoring Unit: (Implied) The core logic handling detection and alerting.
- Mobile Application: For user interaction.
- Wearable Physical Controller: A device (e.g., a watch) for continuous monitoring and fall detection.

S.2 Functionality

  • Continuous Monitoring: The system continuously monitors the home environment for defined events.
  • Automatic Dispatch: The system selects the specific emergency service based on the event type (e.g., fire department for fire events).
  • Visual Verification Integration: The system accesses security camera feeds to verify alerts.

S.3 Interfaces

The system provides the following interfaces for users:
- Mobile App: A primary interface for user interaction.
- Wearable Controller: A physical interface (watch) designed to ensure continuous monitoring even if the user is incapacitated (e.g., falls).

S.4 Detailed usage scenarios

S.5 Prioritization

  • Critical: Immediate alerting for medical emergencies.
  • High: 5-second response for fire detection.
  • High: 10-second response for intrusion detection.

S.6 Verification and acceptance criteria

  • Response Time Verification: The system shall be tested to ensure it meets the 5s (fire), 10s (intrusion), and immediate (medical) thresholds.
  • Reliability Verification: The system must demonstrate <1% false alarm rate over a year.
  • Availability Verification: The system must be verified for 100% uptime.
