G. Goals

G.1 Contexte et Objectifs globaux

G.1.1 Transition vers le "Docs as Code"

L'organisation souhaite abandonner l'édition manuelle de documents de traitement de texte (type Word) pour adopter une approche "Docs as Code". [cite_start]Cela permet de traiter la documentation technique avec la même rigueur que le code source : versionnage via Git, travail collaboratif et traçabilité des modifications[cite: 16].

G.1.2 Automatisation du standard PEGS

Le projet doit fournir un outil capable de générer automatiquement un document de spécifications conforme à la structure PEGS (Project, Environment, Goals, System). [cite_start]L'objectif est de garantir que la documentation finale respecte toujours la structure et la mise en forme validées par l'organisation, sans effort de mise en page manuel[cite: 8, 16].

G.2 Bénéfices attendus

  • Versionnage : Suivi des modifications via Git.
  • Uniformité : Tous les documents auront le même style CSS.
  • Rapidité : Génération instantanée (moins de 1 seconde).

E. Environment

E.1 Glossaire

  • Markdown : Langage de balisage léger utilisé pour le contenu.
  • PEGS : Méthodologie de documentation (Project, Environment, Goals, System).
  • Jinja2 : Moteur de template utilisé par Python.

E.2 Contraintes Techniques

  • Le script doit fonctionner sous Python 3.8+.
  • Aucune connexion internet ne doit être requise pour la génération.
  • Le fichier HTML final doit être lisible sur Chrome et Firefox.

S. System

S.1 Composants

Le système est composé de trois modules : 1. Reader : Scanne les dossiers et lit les fichiers .md. 2. Converter : Transforme le Markdown en HTML. 3. Builder : Assemble le tout dans le template final.

S.2 Fonctionnalités

  • Assemblage : Concaténation des fichiers dans l'ordre G -> E -> S -> P.
  • Table des matières : Génération automatique des liens cliquables (ancres).
  • Style : Injection automatique du fichier CSS dans le HTML (pas de fichier CSS externe).
